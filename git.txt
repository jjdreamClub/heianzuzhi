git 	不管做什么开发都需要

git 	分布式版本控制系统(SVN 集中版本控制系统), 合代码

集中式和分布时的最大区别,在于分支的概念

git,svn  回滚

隐藏文件,一般是某一应用或者系统的配置文件,如果手动修改,或者删除,可能导致应用不可用,或者系统损坏;


git  主要多人协同开发的时候需要使用,那么独立开发可以不使用,git回滚相当于拷贝一个项目,保证,你在的项目损坏的时候,还有一个备份可用;


个人开发使用方式:
1.选中项目文件夹
2.git init 使用git初始化项目
3.可以进行必要的开发;
4.小模块开发完成,并没有问题,需要提交;
	--git status  查看git项目的状态
	--git diff  查看文件修改的内容,但是不可见新文件			
5.git add 文件名 ,文件名是你需要提交的代码文件名 或者直接gitadd . 将所有的改变文件进行包含
6.git commit -m	'提交信息' 将你的修改提交到本地仓库
	--git log  查看日志;
	--git log --pretty=oneline  将log日志,一条线是在一行,只显示版本号和注释信息(一般当我们一天代码提交次数比较多试试用这个方式获取日志)

	--git reset --hard 版本号, 
	--gitreset --hard	HEAD^ 
	一个托字符代表 一个版本当项目开发过程中,出现了问题解决不了时,需要回到上一次代码,可以使用回滚
